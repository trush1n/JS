Итерация с помощью JavaScript Do...While Loops
Следующий тип цикла, который вы изучите, называется циклом do...while. Он называется do...whileциклом, потому что он сначала выполнит doодин проход кода внутри цикла, независимо от того, что происходит, а затем продолжит выполнение цикла, whileуказанное условие которого оценивается как true.

const ourArray = [];
let i = 0;

do {
  ourArray.push(i);
  i++;
} while (i < 5);
Пример выше ведет себя подобно другим типам циклов, и результирующий массив будет выглядеть как [0, 1, 2, 3, 4]. Однако, что отличает этот цикл do...whileот других циклов, так это то, как он ведет себя, когда условие не выполняется при первой проверке. Давайте посмотрим на это в действии. Вот обычный whileцикл, который будет выполнять код в цикле до тех пор, пока i < 5:

const ourArray = []; 
let i = 5;

while (i < 5) {
  ourArray.push(i);
  i++;
}
В этом примере мы инициализируем значение ourArrayпустым массивом, а значение i— 5. Когда мы выполняем whileцикл, условие оценивается как, falseпоскольку iне меньше 5, поэтому мы не выполняем код внутри цикла. Результатом является то, что ourArrayв итоге к нему не будет добавлено никаких значений, и он все еще будет выглядеть так, как []когда весь код в примере выше завершит выполнение. Теперь взгляните на do...whileцикл:

const ourArray = []; 
let i = 5;

do {
  ourArray.push(i);
  i++;
} while (i < 5);
В этом случае мы инициализируем значение i5, как и в случае с whileциклом. Когда мы переходим к следующей строке, нет условия для оценки, поэтому мы переходим к коду внутри фигурных скобок и выполняем его. Мы добавим один элемент в массив, а затем увеличим его, iпрежде чем перейдем к проверке условия. Когда мы наконец оценим условие i < 5в последней строке, мы увидим, что iтеперь оно равно 6, что не проходит условную проверку, поэтому мы выходим из цикла и все готово. В конце приведенного выше примера значение ourArrayравно [5]. По сути, do...whileцикл гарантирует, что код внутри цикла будет выполнен хотя бы один раз.
