На этот раз мы узнаем о трех полезных методах объектов Number: toFixed(), toExponential()и toPrecision(). Их цель — преобразовывать числа в строки, а разница между ними и toString() заключается в следующем:

NumberObject.toFixed(n)

toFixed()принимает 1 параметр n, возвращает строковое представление NumberObject. Он не использует экспоненциальную запись. после десятичной точки зафиксировано n цифр. При необходимости число будет округлено или использовано "0", чтобы достичь указанной длины. Если number больше le+21, метод вызывает только NumberObject.toString() и возвращает строку, представленную экспоненциальной записью.

NumberObject.toExponential(n)

toExponential()принимает 1 параметр n, возвращает строковое представление NumberObject, используя метод экспоненциального подсчета. то есть, есть одна цифра до десятичной точки и n цифр после десятичной точки. Десятичная часть числа округляется. при необходимости, с 0 для составления указанной длины.

NumberObject.toPrecision(n)

toPrecision(n)принимает 1 параметр n, возвращает строковое представление NumberObject, содержит n допустимых чисел. Если num достаточно большой, чтобы включить всю целую часть NumberObject, то возвращаемая строка будет использовать метод подсчета точек. в противном случае используется метод экспоненциального подсчета.

Простое введение абстрактно, нелегко понять. Здесь мы используем примеры, чтобы объяснить разницу между ними:

//example 1
var num=111;
var a=num.toFixed(), 
    b=num.toExponential(), 
    c=num.toPrecision()
console.log([a,b,c])   //put them to an array, we can see the quotes
//output: [ '111', '1.11e+2', '111' ]
Мы можем видеть, что если параметр n опущен, aвозвращается cстрока с номером исходного состояния; bвозвращается строка с использованием метода экспоненциального подсчета, строка содержит максимально возможный десятичный знак.

//example 2
var num=111.11;
var a=num.toFixed(1), 
    b=num.toExponential(1), 
    c=num.toPrecision(1)
console.log([a,b,c]) 
//output: [ '111.1', '1.1e+2', '1e+2' ]
Мы видим, aвозвращает строку с 1 десятичным знаком, используя метод подсчета с фиксированной точкой; bвозвращает строку с 1 целым числом и 1 десятичным знаком, используя метод экспоненциального подсчета; cтакже использует метод экспоненциального подсчета, но возвращает строку с 1 целым числом.

//example 3
var num=111.11;
var a=num.toFixed(6), 
    b=num.toExponential(6), 
    c=num.toPrecision(6)
console.log([a,b,c]) 
//output: [ '111.110000', '1.111100e+2', '111.110' ]
Мы видим, aвозвращает строку с 6 десятичными знаками (составляет «0»), используя метод подсчета с фиксированной точкой; bвозвращает строку с 1 целым числом и 6 десятичными знаками (составляет «0»), используя метод экспоненциального подсчета; cвозвращает строку с 3 целыми числами и 3 десятичными знаками (составляет «0»), возвращает целые и десятичные числа в строке, в общей сложности 6 цифр.

Одним словом, параметр n функций toFixed() и toExponential() используется для цифровых дробных частей; параметр n функции toPrecision() используется для всего числа. Мы можем выбрать их использование в соответствии с различными потребностями.

Иногда наша конечная цель не в том, чтобы использовать эти методы для преобразования числа в строку. Нам нужно использовать значение после изменения. Что нам делать?

Нам нужно изучить два (или три) очень полезных глобальных метода: parseInt()и parseFloat(). Они могут помочь вам вернуть строку в число. Посмотрите пример, чтобы научиться их использовать:

//example 4
var num=111.1234;
var s=num.toFixed(2)   //s=="111.12"
var a=parseInt(s),
    b=parseFloat(s),
    c= +s
console.log([a,b,c]) 
//output: [ 111, 111.12, 111.12 ]
Мы видим, use parseInt(s)может преобразовать s в целое число; use parseFloat(s)может преобразовать s в десятичное число; The '+'— это упрощенная форма the parseFloat()(когда он используется перед строкой).

Задача
Кодирование в функции howManySmaller, функция принимает 2 параметра: arrи n. arr— десятичный массив. n— десятичное число.

Первая задача: пусть все элементы массива сохраняют два знака после запятой (не нужно преобразовывать число n).

Вторая миссия: Обход arr, подсчет количества элементов, меньших n, и возврат его.

например:

howManySmaller([1.234,1.235,1.228],1.24) should return 2
howManySmaller([1.1888,1.1868,1.1838],1.19) should return 1
howManySmaller([3.1288,3.1212,3.1205],3.1212) should return 2
